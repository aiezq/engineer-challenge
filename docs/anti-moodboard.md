# Anti-moodboard

Локальный substitute для anti-moodboard-части submission.

## Чего сознательно избегали
- Перегруженная web3-эстетика с кислотными градиентами
- Темные экраны “по умолчанию” только потому, что это crypto-тема
- Перенасыщенные glassmorphism-слои и декоративные свечения
- Маркетинговый стиль вместо рабочего интерфейса
- Анимации, отвлекающие от auth и dashboard flow

## Почему это важно
- Auth-модуль должен выглядеть надежно, а не хайпово
- В challenge важнее архитектура и operability, чем визуальный шум
- Спокойный UI лучше поддерживает доверие к security-sensitive сценариям
